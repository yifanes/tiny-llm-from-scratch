大语言模型通过预测下一个 token 来学习语言规律。
训练数据是模型学习的原始材料。
训练数据通常来自文本、代码、问答、对话和文档。
在这个项目中，我们先使用很小的中文教学语料。
我们的目标不是训练一个聪明的模型，而是打通完整链路。
模型看到前面的 token，然后预测下一个 token。
tokenizer 负责把文本转换成数字。
字符级 tokenizer 会把每个字符当成一个 token。
词表记录字符和 token id 的对应关系。
训练前，文本会被编码成 token id 序列。
一个模型不能直接理解文字，它只能处理数字。
输入序列和目标序列通常相差一个位置。
如果输入是我爱中，那么目标可能是爱中国。
GPT 的训练目标是 next token prediction。
next token prediction 的意思是预测下一个 token。
Transformer 是现代大语言模型常用的基础结构。
decoder-only Transformer 常用于 GPT 模型。
attention 机制让模型关注上下文中的不同位置。
causal mask 用来阻止模型看到未来的 token。
如果模型训练时看到未来答案，训练目标就失效了。
embedding 把 token id 映射成向量。
position embedding 让模型知道 token 的位置。
LayerNorm 用来稳定神经网络的训练。
MLP 是 Transformer block 中的前馈网络。
residual connection 可以帮助深层网络训练。
loss 用来衡量模型预测和正确答案的差距。
cross entropy loss 常用于语言模型训练。
反向传播会根据 loss 调整模型权重。
AdamW 是训练 Transformer 时常用的优化器。
checkpoint 是训练过程中保存的模型状态。
权重文件保存了模型学到的参数。
config 文件记录模型结构，例如层数、头数和隐藏维度。
tokenizer 文件记录词表和编码规则。
推理引擎负责加载权重并生成文本。
推理阶段通常是一 token 一 token 地生成。
模型每生成一个 token，都会把它接到上下文后面。
temperature 会影响采样的随机性。
temperature 越高，生成越随机。
temperature 越低，生成越保守。
top-k sampling 只从概率最高的 k 个 token 中采样。
top-p sampling 会从累计概率达到 p 的 token 集合中采样。
KV cache 用来缓存 attention 中的 key 和 value。
KV cache 可以减少推理时的重复计算。
OpenAI API 风格接口只是服务协议，不是模型本身。
chat completions 接口通常接收 messages 字段。
messages 字段由 user、assistant 和 system 等角色组成。
本项目会实现一个最小的 chat completions 接口。

问题：什么是大语言模型？
回答：大语言模型是一种通过大量文本学习语言规律的神经网络。
问题：什么是训练？
回答：训练是用数据不断调整模型权重的过程。
问题：什么是验证集？
回答：验证集用于观察模型在未直接训练数据上的表现。
问题：为什么要拆分训练集和验证集？
回答：因为我们需要检查模型是否只是记住训练文本。
问题：为什么第一版使用小数据？
回答：因为小数据更容易理解、调试和复现完整链路。
问题：为什么第一版使用字符级 tokenizer？
回答：因为字符级 tokenizer 简单透明，适合学习。
问题：为什么模型效果不会很好？
回答：因为模型很小，数据也很少，它只能学到有限模式。
问题：这个项目的成功标准是什么？
回答：成功标准是完整跑通数据、训练、导出、推理和 API。
问题：什么是自研推理引擎？
回答：自研推理引擎是自己实现加载权重和 forward 计算的程序。
问题：为什么要对齐 PyTorch 和 Numpy 输出？
回答：因为这可以验证推理引擎的计算是否正确。
问题：什么是 logits？
回答：logits 是模型在 softmax 前输出的原始分数。
问题：什么是 softmax？
回答：softmax 会把 logits 转换成概率分布。
问题：什么是 batch？
回答：batch 是一次送入模型训练的多个样本。
问题：什么是 block size？
回答：block size 是模型一次能看到的最大上下文长度。
问题：什么是 epoch？
回答：epoch 表示模型完整看过一遍训练数据。
问题：什么是 learning rate？
回答：learning rate 控制每次更新权重的步长。
问题：为什么需要 README？
回答：README 用来说明项目目标、运行方式和学习路线。
问题：为什么要每天提交？
回答：每天提交可以记录学习过程，也方便回顾每一步变化。
