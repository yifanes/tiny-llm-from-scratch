# Day 001 - Project Start

Date: 2026-07-06

## Goal

启动一个从零实现 tiny LLM 的长期学习项目。

这个项目的重点不是模型效果，而是完整理解大模型系统链路：

- 训练数据如何变成 token
- token 如何进入 Transformer
- 训练过程如何得到权重
- 权重文件如何被独立推理引擎加载
- 推理为什么是一 token 一 token 生成
- KV cache 为什么能提升推理速度
- OpenAI API 风格接口和模型本身是什么关系

## Decisions

- 使用公开 GitHub 仓库持续记录学习过程
- 第一版模型控制在 8GB GPU 可训练范围内
- 第一版 tokenizer 使用字符级 tokenizer
- 第一版推理引擎使用 Numpy 实现
- 第一版 API 使用 OpenAI chat completions 的最小兼容格式

## Next Step

实现 Phase 1：

1. 实现字符级 tokenizer
2. 生成 train/val token 文件
3. 检查 token 编码和解码是否一致

## Update

已确定 v0 数据策略，并创建 `data/raw.txt`。

v0 数据是手写中文教学语料，包含普通短文、问答格式和用户/助手格式。它的目标不是覆盖大量知识，而是帮助 tiny GPT 在小规模训练中学会固定格式和少量大模型概念。

## Tokenizer

已实现字符级 tokenizer 和数据准备脚本 `data/prepare.py`。

脚本会：

1. 读取 `data/raw.txt`
2. 统计所有不同字符
3. 为每个字符分配 token id
4. 保存 `data/tokenizer.json`
5. 把文本编码为 token ids
6. 按 90/10 切分 `data/train.bin` 和 `data/val.bin`
7. 检查 `decode(encode(text)) == text`
